## 3. Usage Guidelines

1. **Choosing an Integration**  
   - Identify the platform you are using (e.g., GitHub, Confluence, VS Code).  
   - Search the table above for the integration that matches your environment.  
   - Verify whether native support (`✅`) is available; if not, look for community plugins.

2. **Installation**  
   - Most integrations are available as plugins, extensions, or npm packages.  
   - Follow the platform‑specific installation instructions (usually found in the plugin’s README or documentation).

3. **Configuration**  
   - Mermaid diagrams are typically written in Markdown code blocks:  
     ```markdown
     ```mermaid
     graph TD;
         A-->B;
         B-->C;
     ```
     ```  
   - Some integrations expose additional configuration options (e.g., theme, icons).  
   - Refer to the integration’s documentation for advanced settings.

4. **Rendering**  
   - In editors or IDEs, diagrams are rendered live or on save.  
   - In static site generators, diagrams are converted to SVG or PNG during the build process.

5. **Troubleshooting**  
   - Ensure Mermaid.js is loaded (most integrations bundle it).  
   - Check console logs for errors.  
   - Verify that the diagram syntax is correct; use the Mermaid Live Editor for debugging.
